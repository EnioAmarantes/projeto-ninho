# Architecture Decision Records (ADRs)

Esta pasta contém os **Architecture Decision Records (ADRs)** do Projeto Ninho.

Um ADR registra uma decisão arquitetural importante, incluindo seu contexto, as alternativas consideradas e os impactos esperados.

O objetivo é preservar o conhecimento do projeto e facilitar sua evolução ao longo do tempo.

## Estrutura

Cada ADR segue o formato:

* Status
* Contexto
* Decisão
* Consequências
* Alternativas consideradas

## Status possíveis

* Proposed
* Accepted
* Superseded
* Deprecated

Uma decisão nunca deve ser apagada. Caso deixe de ser válida, ela deve ser marcada como **Superseded** e substituída por uma nova ADR.
